## Part 2: OODA Loops MCP System

### Files Created

#### 7. `scripts/embed-loops.ts` (NEW)
- **Purpose**: Build-time script to embed OODA loops from .claude/commands/loops/
- **Lines**: 243
- **Key Features**:
  - Reads all `.md` files in `.claude/commands/loops/`
  - Parses YAML frontmatter with gray-matter
  - Validates file sizes (50KB warning, 100KB hard limit)
  - Detects duplicate loop names across categories
  - Generates TypeScript catalog with metadata
  - Exports `LOOPS_CATALOG` constant

**Algorithm**:
1. Scan `.claude/commands/loops/` directory
2. For each category folder â†’ read .md files
3. Check file size (fail if >100KB)
4. Parse frontmatter (fallback to defaults if missing)
5. Build catalog object
6. Generate TypeScript with helper functions

#### 8. `src/claude-folder-integration.ts` (NEW)
- **Purpose**: Integration with .claude/ folder for usage analytics
- **Lines**: 293
- **Key Features**:
  - Detects `.claude/` folder in working directory or parent
  - Records loop accesses to `loop-usage.jsonl` (atomic append)
  - Aggregates metrics with 3 triggers:
    - On server startup (synchronous)
    - Every 1000 accesses (async background)
    - On explicit refresh (on-demand)
  - Generates `hot-loops.json` (top 10 by usage)
  - Generates `workflow-metrics.json` (detailed stats)
  - Graceful degradation when `.claude/` not present

**Class**: `ClaudeFolderIntegration`

**Methods**:
```typescript
constructor(workingDir: string, logger: Logger)
hasClaudeFolder(): boolean
getClaudePath(): string | null
recordLoopAccess(loopUri, sessionId, mcpSessionId?): Promise<void>
getHotLoops(): Promise<HotLoops | null>
aggregateMetrics(): Promise<WorkflowMetrics | null>
initialize(): Promise<void>
```

## Architecture Changes

### New Patterns Introduced

#### 1. Codebase as Learning Substrate
- `.claude/thoughtbox/` stores usage analytics
- Version-controlled learning (not server-side state)
- Thoughtbox = measurement, not memory

#### 2. Three-Tier Token Optimization
- Tier 1: Workflow prompts (3-5K tokens)
- Tier 2: Loop metadata (500 tokens/loop)
- Tier 3: Full content (5K tokens/loop)
- **Result**: 92% token reduction vs naive composition

#### 3. Build-Time Embedding Pattern
- Similar to existing `embed-templates.ts`
- Read source â†’ parse â†’ validate â†’ generate TypeScript
- No runtime file I/O
