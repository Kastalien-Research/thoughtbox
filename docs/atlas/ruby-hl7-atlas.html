ruby-hl7

A Ruby library for parsing and generating HL7 2.x healthcare messages. Wraps the Health Level 7 messaging standard — the dominant wire format for exchanging clinical and administrative data between hospital information systems — in an idiomatic Ruby object model with a segment DSL, hierarchical parent-child relationships, MLLP transport encoding, and batch-message support.

Key Abstractions & Design Patterns

Architecture
λ
Declarative Field DSL (add_field)
Each concrete segment subclass declares its fields via a class-level DSL. The macro generates named reader/writer methods, registers the field index in a class-level @fields hash, and optionally wraps both read and write through a user-supplied block for validation or type conversion. Auto-incrementing field indices eliminate off-by-one errors common in raw array manipulation.
class HL7::Message::Segment::PID < HL7::Message::Segment weight 1 has_children %i[NK1 NTE PV1 PV2] add_field :patient_id # idx auto = 2 add_field :patient_dob do |val| convert_to_ts(val) # TS format on read+write end add_field :admin_sex do |sex| sanitize_admin_sex!(sex) # validates F|M|O|U|A|N|C end end # Usage pid = msg[:PID] pid.patient_dob # => "19800101" (HL7 TS string) pid.admin_sex = "X" # => raises InvalidDataError
↕
Weight-based Segment Ordering
Every segment class declares an integer weight via a class-level DSL. The weight is stored in the singleton class so subclasses inherit and can override it independently. Segment#<=> implements Comparable by weight diff, allowing Array#sort on a message's segment list. MSH is always first (weight -1). Default is 999 (last). The ORC → OBR → OBX chain uses 88 → 89 → 90 to maintain ordering after dynamic insertion.
class HL7::Message::Segment::ORC weight 88 # Common Order — before OBR end class HL7::Message::Segment::OBR weight 89 # Observation Req — before OBX end def <=>(other) diff = weight - other.weight return -1 if diff.positive? return 1 if diff.negative? 0 end
⊞
Dynamic Child Segment Trees (has_children)
SegmentListStorage#has_children dynamically defines three instance methods on the calling segment class at class evaluation time: children (lazy-init array with a custom <<), child_types, and accepts?(name). The custom << on the children array also propagates the child into the parent HL7::Message's root index, so segment lookup by symbol name works across the hierarchy.
class HL7::Message::Segment::OBR has_children %i[NTE OBX SPM] end # Parser builds the tree automatically: obr = msg[:OBR] obr.has_children? # => true obr.accepts?(:OBX) # => true obr.children # => [OBX, OBX, SPM, NTE] # Adding a child also registers it in msg: obr.children << new_obx msg[:OBX] # => [new_obx, ...]
✱
Open-World Segment Resolution
SegmentGenerator#get_segment_class performs a constant lookup via HL7::Message::Segment.constants.index(segment_name.to_sym). If found, the typed class is instantiated via eval("HL7::Message::Segment::#{seg_name}"). For any unknown segment type (Z-segments, proprietary extensions), it silently falls back to Segment::Default which stores all fields as numbered elements. This guarantees lossless round-tripping of any conformant HL7 message regardless of whether the gem knows the segment type.
def get_segment_class seg = HL7::Message::Segment if seg.constants.index(@seg_name.to_sym) eval("HL7::Message::Segment::%s" % @seg_name) else HL7::Message::Segment::Default # lossless fallback end end # Z-segment round-trip: msg = HL7::Message.new("MSH|...\rZPD|custom|data") zpd = msg.detect { |s| s.e0 == "ZPD" } zpd.e1 # => "custom"
⇄
MLLP Transport Framing
The Minimal Lower Layer Protocol is the standard TCP transport wrapper for HL7 2.x. The library handles both directions: parse_string strips the MLLP envelope from inbound messages using a regex on the VT (\x0B) and FS (\x1C) byte markers, while to_mllp wraps outbound HL7 text with the same framing. The proxy server example demonstrates using this in a real TCP listener.
# Inbound: strip MLLP framing def parse_string(instr) if /\x0b((:?.|\r|\n)+)\x1c\r/ =~ instr post_mllp = $1 # strip bytes end split_by_delimiter(post_mllp, @delimiter.segment) end # Outbound: add MLLP framing def to_mllp "\v" + to_hl7 + "\u001C\r" end # Wire format: # 0x0B | MSH|...\rPID|...\r | 0x1C 0x0D
◆
Dual-Mode Field Access via method_missing
Segment#method_missing provides two orthogonal access modes on top of the named DSL accessors. Named symbols (e.g., :patient_name) route through the fields hash. Index symbols matching e[0-9]+ (e.g., :e5) route directly to element position. This enables generic segment traversal code without knowing field names, while typed code can use readable accessors. Both modes respect the write-path validation blocks.
def method_missing(sym, *args) base = sym.to_s.delete("=").to_sym if self.class.fields.include?(base) # named field path elsif /e([0-9]+)/ =~ base.to_s base = $1.to_i # positional path else super end sym.to_s.include?("=") ? write_field(base, args) : read_field(base) end

Extensibility Architecture

Open/Closed Design
Extension Points
  • Custom Segment Types (Z-segments)
    Subclass HL7::Message::Segment, use add_field DSL, declare weight and optional has_children. The segment auto-registers via constant lookup — no registration step needed. Place in any require-able file; load before use.
  • Field Validation / Conversion Blocks
    add_field :name do |val| ... end attaches a proc executed on both read and write. Used for type coercion (timestamps via convert_to_ts), value validation (sanitize_admin_sex!), and computed fields.
  • Dynamic Segment Definitions at Runtime
    The spec suite (dynamic_segment_def_spec.rb) demonstrates calling add_field on existing segment classes at runtime to add fields to deployed gems without forking. Enables vendor-specific field extensions without patching the gem.
  • HL7::Configuration
    HL7.configure { |c| c.empty_segment_is_error = false } controls whether empty segment strings raise EmptySegmentNotAllowed or are silently skipped. Easily extended with additional parser knobs without breaking the API.
  • Block-form Message Constructor
    HL7::Message.new { |msg| msg << msh; msg << pid } allows builder-pattern message construction. Inside the block, method_missing on the binding context is shimmed to delegate to the segment, enabling a fluent DSL for message assembly.
  • Child Type Registration (add_child_type)
    SegmentListStorage#add_child_type allows appending additional acceptable child segment types to an existing segment class post-definition, enabling incremental extension of the parent-child relationship graph without redefining the whole class.

Notable Implementation Details

Expert Notes
Delimiter Mutability During Parse

The MSH segment encodes the delimiters it uses (MSH[3] = element sep, MSH[4] = item sep). The parser reads these from the first MSH encountered and mutates the Delimiter value object in-place. This means the same HL7::Message instance can correctly parse messages from different systems with different delimiter choices. The delimiters are per-message, not global.

Segment Stack vs. Root Array

During parsing, a segment_stack local array (not stored on the message) tracks the current depth. Each new segment either becomes a child of the stack top (if accepted) or pops the stack until it finds an ancestor that accepts it, or becomes a new root. This implements a simple pushdown automaton for the HL7 message grammar without a formal grammar or recursive descent parser.

JRuby CR/LF Normalization

JRuby's heredoc handling converts \r to \n in string literals, breaking the HL7 segment delimiter assumption. MessageBatchParser#clean_batch_for_jruby detects the presence of \n and does a full String#tr to restore CR characters before splitting — but only for batch parsing, adding a copy only when required.

Singleton Class Metaprogramming

Segment.weight and SegmentFields.add_field store per-class state (weight, field definitions) in the singleton class via module_eval with an ivar (@my_weight, @fields). This ensures each subclass has its own independent field registry without polluting parent class state — a critical correctness requirement for the segment DSL.

Block Binding Shim in Segment Constructor

The block-form Segment.new { |seg| seg.field = val } installs a temporary method_missing on the caller's binding object (eval("self", blk.binding…)) that delegates to the segment instance. This lets block code access the segment as if it were self. The method_missing is removed after the block executes. An acknowledged potential namespace pollution issue is documented in a TODO comment.

Children Array Self-registration

The dynamically-defined children method on segment instances returns a special array whose << method is overridden to walk up the segment_parent chain to find the owning HL7::Message and also appends the child to the message's root @segments list. This dual-registration ensures that msg[:OBX] finds OBX segments regardless of whether they were added as children of OBR or directly to the message.
