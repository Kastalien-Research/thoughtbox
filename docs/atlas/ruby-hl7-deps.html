method_missing Dispatch Chain
How field access and mutation is resolved at runtime — the complete resolution path
Field Access: seg.sending_app / seg.e2 / seg.nonexistent
1
User calls a method on a Segment instance
e.g., msg[:MSH].sending_app or seg.e4 or seg.foo = "bar". Ruby first checks for a real method defined on the object/class. Named field methods like sending_app are generated by add_field via class_eval at class load time — these resolve without hitting method_missing.
2
method_missing(sym, *args, &blk) in segment.rb:97
Called only when no explicit method exists. Extracts base_str = sym.to_s.delete("=") to strip assignment suffix. Constructs base_sym = base_str.to_sym.
3
Branch A: self.class.fields.include?(base_sym)?
Checks the class-level @fields hash (maintained by SegmentListStorage module singleton). If the symbol is a registered field alias, route to read_field/write_field with that symbol.
4
Branch B: /e([0-9]+)/ =~ base_str → numeric element access
Matches e0, e1, e15, etc. Sets base_sym = $1.to_i (an Integer). This routes to read_field/write_field with an integer index directly into @elements array.
5
Branch C: neither → super (raises NoMethodError)
Falls through to super, which raises NoMethodError. Spec confirms: msg[:MSH].does_not_really_exist_here raises NoMethodError.
6
write_field(base_sym, args) or read_field(base_sym)
Determined by sym.to_s.include?("=") for writes, or whether args are present (non-zero args = write). Routes into SegmentFields module:
read_field: looks up idx from field_info, reads @elements[idx], optionally applies field_blk lambda (the conversion/validation block passed to add_field).
write_field: same idx lookup, pads @elements if needed, applies field_blk, stores value.to_s.
7
field_blk application (validation / conversion)
Each add_field block is run on both read AND write. Examples:
convert_to_ts(value) — converts Time/DateTime/Date to HL7 timestamp string via TimeFormatterHelper.
sanitize_admin_sex!(sex) — validates against /^[FMOUANC]$/, raises HL7::InvalidDataError on bad values.
These blocks are stored in @fields[sym][:blk].
8
Block-based initializer method_missing (special path)
When Segment.new { |seg| seg.foo = "bar" } is used, a special METHOD_MISSING_FOR_INITIALIZER heredoc is evaled into the block binding. This temporarily installs a method_missing on the calling context that delegates to __seg__ (the segment instance), then removes itself after the block exits. Allows bare method calls inside the block to route to the segment.
SegmentGenerator class resolution dispatch
1
Message#generate_segment(elm, segment_stack)
Creates a new HL7::Message::SegmentGenerator with the raw element string, current stack, and delimiter. Calls valid_segments_parts? — raises EmptySegmentNotAllowed if config says so.
2
SegmentGenerator#get_segment_class
Looks up segment_to_search = seg_name.to_sym. Checks: HL7::Message::Segment.constants.index(segment_to_search). If found: eval("HL7::Message::Segment::#{seg_name}"). If not found: returns HL7::Message::Segment::Default. Ruby 1.8 compatibility: uses string instead of symbol for constants.index.
3
Message#choose_segment_from(segment_stack, new_seg, seg_name)
Walks the stack backwards. For each prior segment, calls has_children? (responds to children?) and accepts?(seg_name) (checks child_types). If a parent accepts this child: appends to parent's children array, sets is_child_segment = true, pushes to stack. If stack exhausted: appends to top-level @segments, registers in @segments_by_name.

Extension Points
How user code extends the library, plugin architecture, and JRuby-specific code paths
Custom Segment Classes
The primary extension mechanism. Users subclass HL7::Message::Segment directly: 
class HL7::Message::Segment::ZZZ < HL7::Message::Segment
  weight 50
  add_field :my_field
  add_field :validated_field do |value|
    raise HL7::InvalidDataError unless valid?(value)
    value
  end
  has_children [:OBX]
end
Once the class is defined, SegmentGenerator#get_segment_class will automatically find it via HL7::Message::Segment.constants. No registration step required — namespace lookup is the registration.
Dynamic segment definition (segment_generator_spec.rb pattern):
# Segments can be defined at runtime
klass = Class.new(HL7::Message::Segment)
klass.add_field :dynamic_field
HL7::Message::Segment.const_set(:DYN, klass)
Configuration Extension
HL7::Configuration is the single configuration object. Extended via the configure DSL: 
HL7.configure do |config|
  # Allow empty segments instead of raising error
  config.empty_segment_is_error = false
end
The configuration object is memoized at HL7.configuration ||= Configuration.new. Thread-safety is not guaranteed (no mutex). The empty_segment_is_error flag is checked inside SegmentGenerator#valid_segments_parts? to decide whether to raise EmptySegmentNotAllowed or return false.
Planned extension points (based on Configuration class structure): additional parser options, custom delimiter configuration, strict mode flags.
Message Batch / MLLP Protocol Extension
HL7::MessageBatchParser is extended onto HL7::Message as a module: 
# Batch parsing (FHS-wrapped batches)
HL7::Message.parse_batch(batch_string) do |msg|
  # each message in the batch
end

# MLLP framing (vertical-tab + FS + CR)
msg.to_mllp  # wraps in \x0b...\x1c\r
HL7::Message.parse(mllp_string) # strips MLLP automatically
The proxy server example (examples/proxy_server.rb) demonstrates reading raw TCP input, parsing as HL7::Message, and re-sending as MLLP via TCPSocket.
JRuby-Specific Code Paths
Two JRuby workarounds are documented in the source:
1. Batch Parser — clean_batch_for_jruby (message_parser.rb:42)
JRuby converts literal \r characters in heredoc strings to newlines. The private method clean_batch_for_jruby(batch) detects this via batch.include?("\n") and returns batch.tr("\n", "\r"). Only triggered for batch parsing where hl7_batch? returns true. The comment notes this does not occur when posts are received with actual CR characters — only in test heredocs.
2. Segment class lookup — Ruby version fork (segment_generator.rb:43)
In get_segment_class: segment_to_search = @seg_name if RUBY_VERSION < "1.9". Ruby 1.8 requires a String for constants.index; Ruby 1.9+ uses Symbols. This guards against JRuby versions running in Ruby 1.8 compatibility mode.
The Gemfile.lock PLATFORMS section lists arm64-darwin-23 and ruby — no explicit JRuby platform, though the code still maintains compatibility.
Block-Based Segment Construction
Segments support a block constructor pattern via the METHOD_MISSING_FOR_INITIALIZER heredoc trick: 
# Block allows bare field assignments
msh = HL7::Message::Segment::MSH.new do |seg|
  seg.recv_app = "ruby hl7"
  seg.recv_facility = "my office"
  seg.processing_id = rand(10000).to_s
end
Implementation: eval(METHOD_MISSING_FOR_INITIALIZER, blk.binding) installs a temporary method_missing into the calling binding that routes to __seg__ (the segment). After the block runs, remove_method :method_missing is called on the singleton class of the block's context. The comment warns this may "pollute the calling namespace permanently" — a known limitation.
add_child_type — Runtime Child Registration
Beyond has_children at class definition time, the SegmentListStorage module exposes add_child_type for runtime extension: 
# Add a new accepted child type at runtime
HL7::Message::Segment::OBR.add_child_type(:ZZZ)
# Now OBR accepts ZZZ as a child segment
If @child_types is already defined (has_children was called), add_child_type appends to the existing array. If not yet defined, it calls has_children([child_type]) which bootstraps the full machinery.
