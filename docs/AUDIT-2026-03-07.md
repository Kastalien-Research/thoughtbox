# Thoughtbox Codebase Audit — 2026-03-07

**Auditor:** Claude Opus 4.6 (with extensive correction from the human)
**Scope:** Full system — persistence, testing, documentation, knowledge graph
**Trigger:** Attempted to demo basic thought-writing workflow; discovered cascading failures

---

## Executive Summary

The core reasoning system (sessions, thoughts, branches, revisions, gateway, cipher, progressive disclosure) is implemented and persists to disk correctly. Two major subsystems — the knowledge graph and the multi-agent hub — are implemented in code but have no source-of-truth documentation, no real tests, and the knowledge graph has a persistence bug preventing disk writes. The test suite is largely mock-based in violation of project rules. Documentation fractured after January 21, 2026, with subsequent work documented only in scattered specs.

---

## 1. What Works

### Core Reasoning Pipeline
- **Gateway routing** (`src/gateway/gateway-handler.ts`): Single `thoughtbox_gateway` tool routes all operations
- **Progressive disclosure**: Stage 0 -> 1 (`start_new`/`load_context`) -> 2 (`cipher`) -> 3 (domain). Enforced per-session.
- **Session/thought persistence** (`src/persistence/filesystem-storage.ts`): Writes JSON files to `~/.thoughtbox/projects/{project}/sessions/`. Atomic writes via temp+rename. Project isolation via `THOUGHTBOX_PROJECT` env var.
- **Real data on disk**: 92 sessions in project-scoped layout, 115 in legacy layout, with manifests, thought nodes, branches, revision metadata.
- **Entrypoint** (`src/index.ts`): `FileSystemStorage` is the default (`THOUGHTBOX_STORAGE=fs`). `InMemoryStorage` only used when explicitly set to `memory`.

### Source-of-Truth Documentation (stale but exists)
- `docs/docs-for-llms/ARCHITECTURE.md` — system overview, diagrams, component flow
- `docs/docs-for-llms/DATA-MODELS.md` — ~40 type schemas
- `docs/docs-for-llms/TOOL-INTERFACES.md` — gateway routing, tool specs, storage interface
- `docs/docs-for-llms/CONFIGURATION.md` — env vars, cipher protocol, appendices

## 2. What's Broken

### 2.1 Knowledge Graph Not Persisting to Disk

**Symptom:** Created 5 entities and 2 relations via `thoughtbox_gateway { operation: "knowledge" }`. The `stats` and `query_graph` calls return correct data. But:
- `~/.thoughtbox/projects/_default/memory/graph.jsonl` is 0 bytes
- `~/.thoughtbox/projects/_default/memory/memory.db` last modified January 30
- No files under `~/.thoughtbox` were modified on March 7

**Code path:** `src/knowledge/storage.ts` line 332 calls `this.appendJsonl(jsonlEntry)` which calls `fs.appendFileSync`. Line 336 inserts into SQLite. Both writes are in `createEntity`. The code is correct — but the running server instance is not hitting disk.

**Impact:** Knowledge graph is ephemeral. All entities lost when server process stops. The entire project memory feature — the thing that lets agents accumulate knowledge across sessions — does not persist.

**Status:** Bug. Root cause not yet identified. The `FileSystemKnowledgeStorage` is instantiated in `server-factory.ts` line 254 with `project: process.env.THOUGHTBOX_PROJECT || '_default'`. Initialization is wrapped in try/catch (lines 260-263) that logs a warning on failure but continues.

### 2.2 Test Suite Mostly Mocked

**Finding:** 28 of 59 test files use `vi.fn()`, `as any` casts, or mock handlers instead of real storage implementations.

**Project rule violated:** Global `~/.claude/CLAUDE.md` says "Mock boundaries, not logic" and "Only mock things that are slow (network, filesystem), non-deterministic (time, randomness), or external services you don't control." Both `InMemoryStorage` and `FileSystemStorage` are available as real implementations.

**The one real-storage test is broken:** `src/gateway/__tests__/branch-retrieval.test.ts` pairs `ThoughtHandler` with real `InMemoryStorage`. All 9 tests fail with "No session created" at line 105. Broken since February 7. Never fixed.

**What this means:** Zero verified test coverage that the core pipeline (write thought -> read thought -> export session) works end-to-end through real storage. Mock-based tests verify routing/wiring only.

## 3. Corrections Made During This Audit

The human had to correct the auditing agent on the following false claims:

1. **"InMemoryStorage is the only storage"** — Wrong. `FileSystemStorage` exists and is the default. `index.ts` line 74 creates it.

2. **"There are four separate storage systems"** — Wrong. There are three: `ThoughtboxStorage` (sessions/thoughts), `KnowledgeStorage` (knowledge graph), `HubStorage` (multi-agent).

3. **"The knowledge graph was designed for DGM self-improvement"** — Wrong. It is project memory for agents using Thoughtbox in any project. DGM specs reference it but do not own it.

4. **"DGM stands for Directed Graph of Models"** — Wrong. Darwin Godel Machine.

5. **"extract_learnings was designed for DGM"** — Wrong. The `@see dgm-specs/` comments are cross-references, not ownership declarations.

6. **"Agents layered DGM commentary on top of the knowledge graph"** — Wrong. Three `@see` comment lines is not a "layer."

7. **"There is no project-level separation"** — Wrong. `THOUGHTBOX_PROJECT` env var scopes both session storage and knowledge storage. Documented in `CONFIGURATION.md`.

8. **"The persistence layer is not doing anything"** — Wrong for sessions (they persist correctly). The knowledge graph persistence bug is real but specific.

9. **Failed to consult source-of-truth docs first.** The docs exist at `docs/docs-for-llms/` and are labeled "Source of truth" in their own headers. The agent read code, grepped randomly, and made claims without checking the documentation that existed specifically to prevent this behavior.
